# OMS AI Triage Study

**"The 2AM Call vs. The 2AM Google"**

Testing whether frontier LLMs safely triage OMS emergencies — and whether triage recommendations change when a doctor asks vs. when a patient asks.

## Quick Start

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Set your OpenRouter API key
export OPENROUTER_API_KEY=sk-or-v1-...
# (or create a .env file with OPENROUTER_API_KEY=sk-or-v1-...)

# 3. Dry run to see all planned experiment runs
python run_experiment.py --dry-run

# 4. Test with a single scenario
python run_experiment.py --model chatgpt --scenarios 1

# 5. Run the full experiment (~135 runs, ~3-4 hours with rate limiting)
python run_experiment.py

# 6. After scoring, run analysis
python analyze.py data/consensus_scores.csv

# 7. Compute inter-rater reliability
python compute_irr.py data/scorer1.csv data/scorer2.csv data/scorer3.csv
```

## Scenario JSON Format

Each scenario in `prompts/` is a self-contained JSON file:

```json
{
  "id": 7,
  "category": "B",
  "name": "Ludwig's angina",
  "correct_triage": "Emergent",
  "has_patient_frame": true,
  "physician_prompt": "I'm an emergency department physician calling for...",
  "patient_prompt": "Ok so I know this is going to sound stupid but...",
  "critical_elements": [
    "Identifies as airway emergency / prioritizes airway assessment",
    "Recommends awake fiberoptic intubation or tracheostomy",
    "..."
  ],
  "dangerous_recs": [
    "Recommending blind nasotracheal intubation",
    "..."
  ]
}
```

To add a new scenario, create `prompts/scenario_26.json` following this format. The runner and analyzer will pick it up automatically.

## Workflow

### Days 1-3: Prep
- Review scenario JSONs in `prompts/` — verify prompts and scoring keys
- Update model strings in `config.py` if newer versions are available

### Days 4-6: Run Experiment
```bash
python run_experiment.py                          # All 135 runs
python run_experiment.py --model chatgpt          # One model at a time
python run_experiment.py --scenarios 1,2,3        # Specific scenarios
python run_experiment.py --frame physician        # One frame only
```

### Days 7-9: Score
1. Import `experiment_outputs_*.csv` into the scoring spreadsheet
2. Distribute scorer sheets to OMS residents
3. All 3 scorers independently score all outputs
4. Merge into Consensus Scores tab → export as CSV

### Days 10-12: Analyze
```bash
python analyze.py data/consensus_scores.csv
python compute_irr.py data/scorer1.csv data/scorer2.csv data/scorer3.csv
```

## Statistical Analyses

`analyze.py` generates all figures and computes:

**Primary:** McNemar's test for physician vs. patient framing effect, framing gap percentages, overall triage accuracy with 95% CIs

**Secondary:** Under-triage rates, safe uncertainty expression, management accuracy, expected calibration error (ECE), Spearman's confidence-accuracy correlation

**Sensitivity:** S1: Caller specialty effect (ED vs. GP), S2: Excluding Scenario 9 (redirect), S3: Category E standalone, S4: Information-seeking behavior, S5: Confidence-accuracy correlation

- **Gemini** — `google/gemini-2.5-pro-preview-05-06`

## Study Design

- **25 scenarios** across 5 categories (A: post-op, B: ED consults, C: dentist referrals, D: patient direct, E: anesthesia emergencies)
- **3 models** × **2 frames** for Categories A-D = 120 runs
- **3 models** × **1 frame** (physician only) for Category E = 15 runs
- **Total: 135 runs**
- **3 independent scorers** with Fleiss' kappa inter-rater reliability
- Category E excluded from paired framing analysis
